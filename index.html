Project Idea:

Overview:

My project idea is a free Android mobile app designed as a tool to help solve Wordle[4] puzzles. It would be developed using Unity & C# as these are both tools & languages I am already proficient in using, thereby saving on time required to learn their use. The project is designed to be scalable, with a minimum viable product being easy to achieve within the set project time, but with room for additional features to be added as time permits. Once completed, the project is planned to be released onto the android app store, potentially with advertisement integration, time permitting.

Motivation:

Given the explosion in popularity that Wordle[4] has experienced, there is a clear demand for tools to help solve difficult puzzles that users may be struggling with. These tools already exist for the most part, both on the web[5] and on mobile[6]. I also felt this project was a good match for my current programming skill level, as in its’ base version it involves fairly simple search/sort algorithm programming from a word list derived from an english dictionary, which I feel offers a good challenge, covering areas that I don’t have experience in, while still being simple enough to learn in the project timeframe.

Description:

The project is designed to be completed in stages, with the minimum viable product being based on the standard set by the competitor apps I’ve researched[5][6]. Because this minimum viable product should be fairly easy to achieve in the given time, I have planned the project to include further stages, each adding key features to build upon existing ones & improve upon the existing competitor apps, offering an increasing level of complexity & difficulty to the project. Below I outline the milestones & associated features planned for the project, intended to be completed in sequential order:

Minimum Viable Product - The app allows the user to input the state of the puzzle, consisting of attempted guesses, blacklisted letters, confirmed but out of position letters, and fully confirmed letters. Using this data, the app searches through an existing imported list of words derived from the one used by Wordle. I will be looking into finding the exact dictionary Wordle uses to generate its puzzles and if possible directly using this. If this isn’t possible, I will be using an open-source word list from an official English dictionary, of which I know there are many available online. With the word list and current puzzle state, the app will narrow down the dictionary word list to a list of possible correct answers. The exact sort/search algorithm to be used is yet to be determined, and this will be an area I will be directing research towards during the beginning stages of the project. The app will then display this list of possible solutions to the user.

Smart Guesses- The app uses statistical analysis to analyze the results from the foundational word search system, to determine which letters are more commonly used in the overall dataset and possible solution set, and use this to make intelligent suggestions for guesses, as opposed to simply listing all the possible solutions like in the minimum viable product stage.

Advertising Integration- The app incorporates mobile advertisements in order to generate some revenue from users. These ads would be presented at the beginning of a new use session, limiting them to a maximum of one per day, as that’s how often the global Wordle puzzles themselves refresh.

Stretch goal/Automation- The app works as a screen overlay on top of the puzzle as you solve it in another window, automatically reading & interpreting the state of the puzzle via an image recognition system. The app then feeds this data back into the existing foundational word search system, and generates suggested guesses in the background, which are then presented to the player either in the form of an overlaid UI element, or as a push notification. This means the user never has to shift back to the app itself once it has been started and manually input the current puzzle state.

Tools & Technologies:

This Wordle Solver app would be built using Unity & C# for reasons already outlined in the overview. My intent is to deploy the app for Android devices. Ideally I would like to be able to create both Android & iOS versions, but without an iOS device I would be unable to test it and therefore would not be able to do any of the important bug testing required on that platform. My testing platform will be a Samsung Galaxy S8. Version control will be handled through Github Desktop. I will also be using Visual Studio code as my IDE during development.

Skills Required:

This project will require knowledge of how to use the Unity game engine, mobile specific development practices & basic C# programming knowledge. I will need to learn how to import and work with an appropriate word list into Unity in order to implement the functionality required to be able to search through the possible word lists. I will also need a good understanding of how to program robust searching algorithms, as the program will be dealing with & sorting through large amounts of text data. I will also require a basic understanding of GitHub & version control software, in this case GitHub Desktop, in order to manage & back up project files, which I have.

Outcome:

If the project is successful, the outcome will largely be dependent on what stage of the project we have reached. If we only achieve the minimum viable product milestone, we will have made an app on par with most others on the market, but without anything to make it stand out, which will severely limit the impact it would have.

However with additional milestones reached, the project will be able to bring new features to the market & hopefully attract more attention & users. A best case scenario for this project would be the completion of all milestones, and a significant acquisition of regular users, generating a mild amount of income in the form of advertising revenue for the team.
